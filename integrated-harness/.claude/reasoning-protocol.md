# 深廣思考協定

任務風險決定思考深度；核准模式只決定授權關卡，兩者互不取代：

- 純唯讀或機械性工作：可在內部合併五階段，只回報結論與驗證證據。
- 一般修改：保留拆解、方案選擇、驗證；沒有實質決策時不強求多方案。
- 高風險、架構、個資、正式環境或不可逆工作：完整依序執行五階段，不得跳過，
  即使政策模式是 `light` 也相同；但完整思考不代表已取得人類授權。

## 1. 拆解

- 重述實際要求並界定不在範圍內的事項。
- 列出技術、維運、資安、成本、人為因素與邊界案例。
- 分別列出已知資訊、缺少資訊及標為 `【假設】` 的假設。

## 2. 探索

- 有實質決策時提出至少兩個本質不同的方案，分別說明機制、優勢、弱點及適用條件。
- 瑣碎或只有單一可行做法時直接說明原因，不為滿足數量製造陪榜方案。

## 3. 深化

- 對最佳候選推進到可執行、可測試的細節。
- 追蹤主要二階後果與故障模式。
- 找出最脆弱的假設並設計驗證方式。

## 4. 對抗式審查

- 高風險工作至少提出兩個真實異議；一般工作至少檢查一個最可能的反例。
- 檢查隱含假設、倖存者偏差、對題目範例過度擬合，以及相關性與因果性混淆。
- 異議成立就修正；不能消除的不確定性明確回報。

## 5. 輸出前驗證

- 逐項對照原始要求與驗收標準。
- 確認事實、`【推論】` 與 `【假設】` 沒有混淆。
- 先講結論，附上驗證證據、取捨與剩餘風險。
- 標示信心程度（高／中／低），並說明哪個關鍵假設若錯誤會改變結論。
