# 將完整編排改為精簡治理政策

日期：2026-07-23

## 決策

`integrated-harness/ORCHESTRATOR.md` 不再教導模型如何分解任務、選擇模型或調度代理。
這些一般工作策略交由 Claude／Codex 平台與模型自行決定。文件只保留人類無法由模型
自行推知、且需要稽核的治理邊界：計畫與核准、外部副作用、敏感資料、修改範圍、
驗收證據、成本門檻與失敗揭露。

`harness/fable-orchestrator-prompt.md` 標示 deprecated，只為既有連結與使用者保留；
產生 `ORCHESTRATOR.md` 提示稿不再是 Harness 的產品功能或建議流程。

## 理由

模型能力與平台原生代理功能持續提升，通用分解與路由教學容易過時、占用 context，
也可能與平台策略衝突。模型能力無法取代人類授權與確定性 hooks，因此治理邊界及
安全關卡繼續保留。
